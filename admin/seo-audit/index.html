2즉시 적용 — 이 PR에 포함

"author":{"@type":"Person","name":"베이비빌리 보험 전문팀","jobTitle":"부모·가족 보험 컨설턴트","worksFor":...}

HIGH Organization sameAs (소셜·커뮤니티 백링크)

베동 커뮤니티·카톡채널 등 외부 신호를 schema에 명시. Knowledge Graph 형성 도움.

MID 5세대-청구-시뮬 FAQPage 추가

유일하게 FAQPage 누락. 청구 시뮬 페이지에 보장률 FAQ 자연스럽게 배치 가능.

3Pillar-Spoke 구조 — 권장사항

현재 구조 분석:

  • ✅ 5세대 실손 — Pillar(완전정리) + Spokes(7개) 정상

옵션 A: 허브를 Pillar화 (이번 PR 적용)

가장 적은 노력. /guide/와 /child/ 허브 페이지에 종합 콘텐츠 섹션 추가하여 자연스럽게 Pillar 역할 부여.

  • 장점: 새 URL 안 늘림, 기존 SEO 자산 활용
  • 단점: 카드 그리드만 있던 허브가 길어짐 (UX 저하 가능)

옵션 B: 새 Pillar 페이지 생성 (권장 — 별도 PR)

5세대처럼 종합 가이드 신규 작성:

  • /guide/태아보험-완전정리/ — 가입시기·보험사·보험료·특약·청구 한 페이지
  • /child/어린이보험-완전정리/ — 재설계·비교·특약·보험료·환급형 한 페이지

예상 효과: "태아보험" 단일 키워드 (월 검색량 ~5만) 상위 노출 도전 가능. 현재는 롱테일 키워드만 잡고 있음.

  • ✅ FAQPage — 질문-답변 추출 용이

5콘텐츠 신호 — 검토 필요

YMYL (Your Money Your Life) 페이지 신뢰도

보험 = 금융·건강 양쪽 YMYL. Google이 가장 까다롭게 평가. 다음 신호 추가 권장:

  • 최종 검수일 표시 (<time datetime=...>) — 정보 최신성
  • 실제 작성자 정보 — "베이비빌리 보험 전문팀" + 경력 명시
  • 출처/참고자료 — 보험사 약관·금감원·보건복지부 자료 인용
  • 법적 고지 — "본 콘텐츠는 정보 제공 목적..." (이미 footer에 있음 ✓)
  • 개인정보처리방침·이용약관 페이지 (현재 없음)

관련 페이지 추가 검토

  • /about/ — 회사·작성자 소개 (E-E-A-T 핵심)
  • /privacy/ — 개인정보처리방침
  • /terms/ — 이용약관
  • /contact/ — 문의·연락처
